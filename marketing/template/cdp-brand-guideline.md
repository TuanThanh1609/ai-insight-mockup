### 2.2 Logo Design Language

**Icon Mark (CDP Circle Symbol):**
- Vòng tròn đồng tâm — tượng trưng cho dữ liệu đa kênh hội tụ về 1 điểm
- Bên trong vòng tròn: chữ **"CDP"** cách điệu — chữ C và P kết nối liền mạch
- Chữ **D** nổi bật với thanh ngang đậm (signature element)
- Phong cách: **Flat, tối giản, đáng tin cậy** — "Digital Curator" aesthetic
- Màu: White trên nền trong suốt → đặt được trên mọi nền

**Lockup ngang (5:1):**
```
[ 🅞🅓🅟 CDP Circle Icon ]          [        CDP        ]
     (~22% width)                         (~78% width)
  Icon mark cách điệu                   Wordmark sans-serif bold
```

**Lockup vuông (1:1 — Mobile/App Icon):**
```
┌──────────────────────────┐
│                          │
│       🅞🅓🅟            │  ← Icon mark tràn đầy vùng vuông
│       (Circle)           │
│                          │
└──────────────────────────┘
```

### 2.3 Color Variants

| Variant | Nền phù hợp | Logo color | Khi nào dùng |
|---------|------------|-----------|--------------|
| `LOGO CDP TRẮNG.png` | `#1A2138` Deep Navy, nền tối | White `#fcf8fb` | Sidebar, nav, footer, dark hero, pitch deck |
| `LOGO CDP XANH.png` | `#FCF8FB` Surface, nền sáng | Vibrant Blue `#0052FF` | Header nav (light page), proposal, print |
| `LOGO CDP MOBILE 1/2.png` | Bất kỳ nền nào | White transparent | App icon, avatar, favicon |

### 2.4 Logo Clear Space

```
Logo ngang (TRẮNG / XANH):
  Min clearance: ½ chiều cao logo — cả 4 cạnh
  Ví dụ: logo cao 20mm → clearance tối thiểu 10mm mỗi cạnh

Logo vuông (MOBILE 1/2):
  Min clearance: ½ chiều rộng logo — cả 4 cạnh
```

### 2.5 Logo Size Restrictions

| Ngữ cảnh | File | Chiều rộng tối thiểu |
|---------|------|---------------------|
| Social media post | `MOBILE 1.png` | 120px |
| Pitch deck slide | `TRẮNG.png` | 200px |
| Landing page header | `TRẮNG.png` hoặc `XANH.png` | 160px |
| Email header | `TRẮNG.png` | 100px |
| App icon (iOS/Android) | `MOBILE 1.png` | 1024×1024px (native) |
| Favicon / Tab | `MOBILE 1.png` | 16×16px, 32×32px |
| Sidebar nav | `TRẮNG.png` | 140px |

### 2.6 Logo DO's & DON'Ts

```
✅ ĐƯỢC:
  ✓ Giữ nguyên tỉ lệ pixel — không co giãn
  ✓ TRẮNG.png → nền tối (Deep Navy, gradient tối)
  ✓ XANH.png → nền sáng (Surface, trắng, giấy)
  ✓ MOBILE 1/2 → app icon, avatar, favicon
  ✓ Đặt MOBILE 1/2 trên nền có độ tương phản đủ (kiểm tra visual)

❌ TUYỆT ĐỐI KHÔNG:
  ✗ Co giãn, bóp méo tỉ lệ
  ✗ Thêm shadow, outline, glow, gradient effect lên logo
  ✗ Đổi màu ngoài 3 variants (TRẮNG / XANH / MOBILE transparent)
  ✗ Đặt XANH.png trên nền tối (contrast quá thấp)
  ✗ Đặt TRẮNG.png trên nền nhạt (contrast quá thấp → dùng XANH)
  ✗ Đặt logo trên nền có pattern/gradient phức tạp (clear space bị phá vỡ)
  ✗ Tách icon CDP Circle ra khỏi wordmark trong lockup ngang
  ✗ Dùng làm favicon (dùng MOBILE 1/2 crop thay vì TRẮNG/XANH ngang)
```

## 4. Typography Quy Chuẩn

### 4.1 Font Stack

```
HIỂN THỊ (Display / Headlines):
  Primary:   Manrope Bold 700
  Fallback:  system-ui, -apple-system, sans-serif
  Sử dụng:   Page titles, hero headlines, dashboard scores

THÂN BÀI (Body / UI):
  Primary:   Inter Regular 400 / SemiBold 600
  Fallback:  system-ui, -apple-system, sans-serif
  Sử dụng:   Descriptions, labels, buttons, data

TIẾNG VIỆT (Vietnamese-specific):
  Primary:   Be Vietnam Medium 500 (body copy trong tài liệu)
  Display:   Be Vietnam Ultra-Bold 800 (headlines tiếng Việt trên slide)
  Fallback:  system-ui, -apple-system, sans-serif

SOCIAL MEDIA (Cho ảnh đăng):
  Display:   Poppins Bold 700
  Headline:  Poppins SemiBold 600
  Body:      Poppins Regular 400
  Condensed: League Spartan Bold (compressed headlines trên ảnh)
```

| Class | Font | Weight | Size | Line-height | Usage |

### 4.3 Vietnamese Typography Rules

```
QUY TẮC BẮT BUỘC:
  1. Luôn dùng font có Unicode tiếng Việt đầy đủ (Manrope, Inter, Be Vietnam)
  2. Letter-spacing mặc định: 0 (không thêm khoảng cách giữa các chữ cái)
  3. Word-break: cho phép break line giữa các syllable tiếng Việt
  4. Label/overline: ALL CAPS + letter-spacing +5% (0.05em)

QUY TẮC BREAK-LINE:
  - Heading dài: ưu tiên break sau dấu phẩy, gạch ngang, hoặc giữa cụm ý
  - Body text: justify hoặc left-align, không hyphenate tiếng Việt

VÍ DỤ HEADLINE BREAK:
  ✗ "Phântích10nhómvấnđề"
  ✓ "Phân tích 10 nhóm vấn đề\ntrong hội thoại khách hàng"

VÍ DỤ LABEL:
  ✓ "CHUYÊN GIA SMAX GỢI Ý"
  ✓ "NHÓM BỆNH"
```

## 5. Photography & Imagery

### 5.1 Photography Style

**Mood & Tone:**
- Professional, premium, trustworthy — NOT stock photo cheesy
- Real people doing professional work — NOT staged corporate handshake shots
- Technology + human warmth — NOT robot/AI dystopia

**Subjects:**
| Nên có | Không nên có |
|--------|-------------|
| Real dashboards / UI mockups | Fake smiling people pointing at screens |
| Data visualization | Stock photo families |
| Vietnamese business contexts | Western office environments |
| Team collaboration | Artificial lighting |
| Product UI screenshots | Abstract 3D renders |

**Color Treatment:**
- Warm tones preferred — phù hợp với Surface palette
- Minimal color grading — giữ màu thực
- Natural light sources

### 5.2 Screenshot & UI Photography

```
QUY TẮC CHỤP MÀN HÌNH:
  ✓ Dùng real product screenshots từ staging/production
  ✓ Crop chính xác vùng cần show
  ✓ Giữ tỉ lệ pixel chính xác (không resize distortion)
  ✓ Thêm subtle shadow để tách khỏi nền

KHI DÙNG TRONG TÀI LIỆU:
  ✓ Đặt trên Surface (#FCF8FB) background
  ✓ Thêm subtle shadow nếu floating
  ✓ Giữ 8px padding xung quanh
  ✓ Không thêm border — dùng tonal shift

KHI DÙNG TRONG ẢNH SOCIAL:
  ✓ Đặt trên dark background (#1A2138) để nổi bật
  ✓ Thêm subtle gradient overlay để blend với background
  ✓ Giữ minimum 48px margin từ edges
```

### 5.3 Illustration Style

**Khi nào dùng illustration thay vì ảnh chụp:**
- Đồ thị/sơ đồ phức tạp cần simplify
- Concept khó chụp thực tế (ví dụ: AI processing)
- Icon illustrations cho feature highlights
- Technical diagrams cho pitch deck

**Style Guide:**
- Flat design — NO 3D renders
- Brand colors only (Deep Navy, Deep Rust, Vibrant Blue, Surface)
- 8px rounded corners consistent
- Subtle shadows (tinted, not black)
- Line weight: 2px stroke minimum for readability

## 7. Spacing & Layout

### 7.1 8px Grid System

**Mọi spacing = bội số của 8px:**

```
4px   → Micro gaps, icon-to-text, dense data tables
8px   → Default component spacing, list item gap
16px  → Card padding, button-to-button gap
24px  → Section internal padding, card group gap
32px  → Card main padding (token 6)
48px  → Section gap (token 8)
64px  → Major section break (token 9)
96px  → Hero section padding (token 11)
128px → Page-level major separation (token 12)
```

### 7.2 Layout Principles

```
QUY TẮC "DIGITAL CURATOR":
  1. Intentional Asymmetry: Khoảng trắng lớn (token 16–24)
     để dẫn mắt về insight chính

  2. Tonal Depth: Không dùng border lines —
     dùng surface shifts (light → slightly darker)

  3. High-Contrast Energetics: Deep Navy nền +
     Vibrant Blue accents = authority + momentum

  4. Bottom-Up Reading: Layout nên guide từ
     summary → detail (điểm → metrics → actions)

LAYOUT RESPONSIVE BREAKPOINTS:
  Mobile  → < 640px  (1 column, stacked)
  Tablet  → 640–1024px (2 columns)
  Desktop → 1024–1440px (3 columns, sidebar)
  Wide    → > 1440px  (max-width container, centered)
```

### 7.3 Content Width Guidelines

| Loại content | Chiều rộng tối đa | Centered? |
|-------------|-------------------|-----------|
| Body text | 720px | Yes |
| Dashboard content | 1440px (full-width) | No |
| Landing page hero | 1200px | Yes |
| Marketing images | 1080px (native) | N/A |
| Slide content | 1920px | N/A |

## 8. Animation & Motion

### 8.1 Motion Principles

```
ĐƯỢC DÙNG:
  ✓ Fade + subtle slide (translate Y 8–16px)
  ✓ Spring easing cho interactive elements
  ✓ Staggered entrance cho lists (50–100ms delay)
  ✓ Progress animations (loading bars, counters)
  ✓ Radar scan effect cho hero sections

KHÔNG DÙNG:
  ✗ Autoplay video backgrounds
  ✗ Blinking / flashing animations
  ✗ Spin/rotate loading indicators (dùng skeleton)
  ✗ Parallax scroll effects quá mạnh
  ✗ Bounce animations trong B2B context
```

### 8.2 Animation Timing

```
Micro-interactions (hover, focus):
  Duration:  150–200ms
  Easing:    ease-out
  Example:   Button hover color change

Element entrance:
  Duration:  300–500ms
  Easing:    ease-out hoặc cubic-bezier(0,0,0.3,1)
  Example:   Card fade-in on scroll

Page transitions:
  Duration:  200–400ms
  Easing:    ease-in-out
  Example:   Modal open/close

Data updates:
  Duration:  600–1000ms
  Easing:    ease-in-out
  Example:   Number counter animation, progress bar fill
```

## 9. Tone of Voice

### 9.1 Voice Framework

**4 chiều Voice:**

```
AUTHORITY ←————————————————————————————————————————————→ FRIENDLY
  "Phân tích AI chính xác 97%"     "Biết bệnh của mình rồi chứ?"
  (Data, precision, confidence)     (Empathy, casual, approachable)

URGENCY ←——————————————————————————————————————————————→ CALM
  "Junk Lead 42%! Hành động ngay!"  "Hồ sơ bệnh án cập nhật mỗi ngày"
  (Alert, action required)           (Continuous, no pressure)

FORMALITY ←————————————————————————————————————————→ CASUAL
  "Chuyên gia Smax gợi ý hành động" "Chuyên gia Smax gợi ý"
  (Product naming)                  (UI labels, shortcuts)

CLARITY ←————————————————————————————————————————————→ BREVITY
  "Thu thập SĐT: 23%  •  Chốt đơn: 18%"  "SĐT 23%, Chốt 18%"
  (Full context)                   (Dashboard, cards)
```

**Vị trí Brand Voice:** Authority + Calm + Formality + Clarity

### 9.2 Voice by Touchpoint

| Dashboard copy | Clarity + Calm | "Điểm sức khỏe: 6.2/10 — Cần cải thiện" |

### 9.3 Từ Vựng Chuẩn

**LUÔN DÙNG:**
```
  ✓ "Khám Bệnh Hội Thoại"          (tên tính năng đầy đủ)
  ✓ "Hồ sơ bệnh án"                (patient record metaphor)
  ✓ "Chuyên gia Smax"               (expert agent)
  ✓ "Lead Nóng / Ấm / Lạnh"        (temperature — không Hot/Warm/Cold)
  ✓ "Nhóm bệnh" / "Hạng mục"       (disease group — không "loại bệnh")
  ✓ "Gợi ý" / "Hành động"          (recommendation)
  ✓ "Điểm sức khỏe"                (health score)
  ✓ "Real-time mỗi ngày"            (daily update)
  ✓ "Cải thiện" / "Tăng / Giảm"   (improvement indicators)
  ✓ "Cần cải thiện"                (severity — không "bệnh nặng" trong UI)
```

**TUYỆT ĐỐI KHÔNG DÙNG:**
```
  ✗ "Bệnh" (một mình)                → "nhóm bệnh" hoặc "hạng mục"
  ✗ "AI" khi chỉ Chuyên gia Smax   → dùng "Chuyên gia Smax"
  ✗ "Bot" / "Robot"                 → "Chuyên gia Smax"
  ✗ "Hot / Warm / Cold" (English)   → "Nóng / Ấm / Lạnh"
  ✗ "Tự động" (chung chung)         → "Auto-reply", "auto-tag", "auto-label"
  ✗ "Tốt nhất thị trường"           → không comparative claim
  ✗ "Hoàn toàn miễn phí!"           → "Miễn phí" (không viết HOA)
  ✗ Emoji spam ( > 3 emoji/post)   → tối đa 1-2 emoji
```

### 9.4 Headline Patterns

```
PATTERN 1 — Problem-Aware:
  "[Pain Point cụ thể]? [Solution]"
  Ví dụ: "Junk Lead 42%? Biết ngay nhóm nào đang kéo tụt."

PATTERN 2 — Transformation:
  "[Before state] → [After state]"
  Ví dụ: "Từ 'không biết' → 'biết chính xác 10 nhóm bệnh'"

PATTERN 3 — Authority Claim:
  "[Số liệu ấn tượng] — [Context]"
  Ví dụ: "10 nhóm bệnh. Real-time mỗi ngày. Hành động cụ thể."

PATTERN 4 — Question-Driven:
  "[Rhetorical question] — [Answer]"
  Ví dụ: "Hội thoại khách hàng đang 'bệnh' ở đâu?"
```
